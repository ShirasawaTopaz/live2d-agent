# 模式管理功能文档

## 1. 功能概述

本文档介绍 Live2oder 应用中新增的"Plan模式"和"编排模式"功能，这些模式用于控制输入框的行为和消息处理方式。

## 2. 模式说明

### 2.1 Plan模式

**功能**：
- 启用后，输入的消息会被标记为 `[Plan模式]`，表示这是一个规划相关的指令
- 主要用于创建、修改和执行任务计划
- 适用于需要AI生成详细执行计划的场景

**使用场景**：
- 复杂任务的分解和规划
- 多步骤操作的协调
- 需要AI生成详细执行计划的情况

### 2.2 编排模式

**功能**：
- 启用后，输入的消息会被标记为 `[编排模式]`，表示这是一个任务编排相关的指令
- 主要用于管理和执行已有的任务计划
- 适用于需要AI协调多个任务执行的场景

**使用场景**：
- 执行已创建的任务计划
- 监控任务执行状态
- 任务之间的依赖管理

## 3. 使用方法

### 3.1 启用/禁用模式

1. 在输入框底部的工具栏中，点击 "Plan模式" 按钮启用或禁用 Plan模式
2. 点击 "编排模式" 按钮启用或禁用 编排模式

**注意**：
- 两个模式可以同时启用，但通常建议一次只启用一个模式
- 模式启用时，按钮会显示高亮颜色：
  - Plan模式：绿色
  - 编排模式：蓝色

### 3.2 输入消息

启用模式后，输入框的占位符会相应变化：
- Plan模式："输入Plan模式指令..."
- 编排模式："输入编排模式指令..."

输入消息后，按 Enter 键或点击 "发送" 按钮发送消息。

### 3.3 消息处理

发送的消息会自动添加模式标记：
- Plan模式：消息前添加 `[Plan模式]` 标记
- 编排模式：消息前添加 `[编排模式]` 标记

这些标记会帮助AI理解消息的意图，从而提供更准确的响应。

## 4. 技术实现

### 4.1 UI组件

- **Plan模式按钮**：QPushButton，可切换状态
- **编排模式按钮**：QPushButton，可切换状态
- **状态管理**：使用布尔变量跟踪模式状态

### 4.2 核心逻辑

1. **模式切换**：
   - 点击按钮触发 `on_plan_mode_toggled` 或 `on_orchestration_mode_toggled` 槽函数
   - 更新内部状态变量
   - 更新按钮样式和输入框占位符

2. **消息发送**：
   - 在 `send_message` 方法中检查当前模式
   - 根据模式添加相应的标记
   - 发送带有标记的消息

### 4.3 状态保存

模式状态在应用运行期间保持，应用重启后会重置为默认状态（禁用）。

## 5. 已知限制

1. **模式冲突**：两个模式同时启用时，Plan模式优先级更高
2. **状态持久化**：模式状态不会在应用重启后保存
3. **UI反馈**：仅通过按钮颜色和占位符变化提供反馈，没有专门的状态指示器

## 6. 代码结构

相关文件：
- `internal/ui/input_box.py`：实现模式切换功能
- `internal/agent/planning/`：模式对应的后端功能实现

## 7. 示例

### 7.1 Plan模式示例

**输入**：
```
创建一个计划来清理系统垃圾文件
```

**实际发送**：
```
[Plan模式] 创建一个计划来清理系统垃圾文件
```

### 7.2 编排模式示例

**输入**：
```
执行清理系统的计划
```

**实际发送**：
```
[编排模式] 执行清理系统的计划
```

## 8. 故障排除

### 8.1 模式不生效

- 检查按钮是否正确点击（应显示高亮颜色）
- 检查输入框占位符是否相应变化
- 检查发送的消息是否包含模式标记

### 8.2 消息标记不正确

- 确保只启用了一个模式
- 检查发送的消息是否正确添加了标记
